# UI Polish: Status Bar, Commands, and Interaction Improvements

**Goal:** Address 8 UI refinements: rename `/container-shell` to `/shell`, replace `← current` with green dot, add icons and diff stats to status bar, improve worktree count format, dynamic name truncation, autocomplete enter behavior, and faster container exit.

## Codebase Context

- **main.go** — Commands list at line 91: `["/branches", "/config", "/container-shell", "/model", "/worktrees"]`. `handleCommand()` at line 1104 routes slash commands. `renderStatusBar()` at line 1384 renders left side (branch + PR) and right side (worktree name + active count). Enter handling at line 696 sends full command text to `handleCommand()` — does NOT check autocomplete matches. `cleanup()` at line 1136 calls `container.Stop()`.
- **container.go** — `Stop()` at line 160 does `docker stop -t 3` (graceful 3s grace) THEN `docker rm -f`. Two sequential Docker commands = slow exit.
- **branchlist.go** — Line 210: `← current` marker in purple `#9B6ADE` for current branch.
- **worktreelist.go** — Line 158: same `← current` marker for current worktree session.
- **modellist.go** — Line 298-300: already uses green `●` (`#6FE7B8`) for active model — this is the target pattern for "current" indicators.
- **statusInfo** struct (line 222): `{Branch, PRNumber, WorktreeName, ActiveCount}`. `fetchStatusCmd()` at line 324 counts active worktrees but doesn't track total count.
- **exec_test.go** — Tests reference `/container-shell` command string and `enterShell()` helper types it.
- **statusbar_test.go** — Tests check for `"2 active"` string format and branch content.

## Phase 4: Dynamic name truncation

- [x] 4a: In `renderStatusBar()`, compute a max width budget for branch name and worktree name based on `m.width`. Reserve space for fixed elements (icons, PR number, diff stats, count, padding). Split remaining space proportionally between branch and worktree names. If a name exceeds its budget, truncate with `…` suffix. This ensures the status bar doesn't overflow on narrow terminals while maximizing displayed text on wide terminals. Add test for truncation behavior at narrow widths.

## Phase 5: Autocomplete enter and faster exit

- [x] 5a: Make Enter trigger first autocomplete match — in the `"enter"` case of modeChat key handling (main.go:696), before calling `handleCommand(val)`, check if the input starts with `/` and if `autocompleteMatches()` returns matches. Add test: type `/sh` and press Enter → should enter shell mode (first match is `/shell`).
- [x] 5b: Faster container exit — in `container.go` `Stop()`, replace the two-step `docker stop -t 3` + `docker rm -f` with a single `docker rm -f` which force-kills and removes in one command. Reduce the context timeout from 15s to 5s. The graceful stop is unnecessary since we don't need to persist container state. Update the comment. Existing `container_test.go` tests should still pass since fakes handle both `stop` and `rm` cases — may need minor adjustments if tests check for `stop` specifically.

## Success Criteria

- `/shell` replaces `/container-shell` — shorter to type, no ambiguity with `/config`
- Green dot `●` marks "current" items consistently across branches, worktrees, and models
- Status bar shows `` icon before branch, `` before worktree name
- Worktree count shows `(2/5)` format only when there are multiple worktrees
- Branch diff stats `+X/-Y` visible next to branch name
- Names truncate with `…` on narrow terminals, expand fully on wide ones
- Pressing Enter with partial slash command (e.g., `/sh`) triggers the first autocomplete match
- Container exit is near-instant (single `docker rm -f` vs graceful stop)
- All existing tests pass (updated as needed)
